## 共通運用（ツール一覧）

目的: 規程本文を変更せずに、使用ツールの一覧・設定の最小限を本別紙で一元管理する。

- 発効時点（effectiveAt）: ＜日時/ブロック高＞
- Version: v0.1
- 改訂日: 2025-09-30
- 改訂者: 
- 変更理由: 初版（shared を単一ドキュメントへ統合）

### ツール一覧・設定（共通｜ミニマム）
- コミュニケーション: Discord, Telegram, Email
- 投票管理: Snapshot（例）, その他EVM互換ツール
- ウォレット: MetaMask 等EVM互換
- フォーム/名簿: Google Form / Spreadsheet
- リポジトリ/記録: GitHub / Cursor / Notion（該当時）
- トークン発行ツール: EVM互換の発行ツール（例: ERC-20/721対応ツール）

### 開示・周知・記録に関わるツール分類（参照用）
- 開示・公開先（閲覧用）
  - DAO総会プラットフォーム（例: Snapshot Space/投票ページの公開）
  - 公式Web/ポータル（ウェブサイト/ブログ/告知ページ）
  - リポジトリ公開領域（例: GitHub `docs/` やリリースノート）
  - ナレッジベース（該当時: Notion 等）
- 通知・周知チャネル
  - Discord 告知チャンネル／役職メンション
  - メーリングリスト（全メンバー配信）
  - Telegram（フォールバック/補完）
- 記録・保存（証跡/監査）
  - メッセージURL／投票ページURLの保全
  - スクリーンショット保管（画像リポジトリ/Notion添付）
  - リポジトリへの文書化（`docs/`、議事録・開示PDF/MD）
  - 変更履歴の管理（コミット/PR/ページ履歴）

### 開示の種別と標準手段（限定/一般）
- 限定開示（議決権保有者向け）：
  - 告知（投票開始前日）: Snapshotの提案/投票ページURL＋Discord告知（役職メンション）。必要に応じてメーリングリスト配信。
  - 議決結果の告知: 同様に限定チャネルで実施（期限は本文で規定）。
  - アクセス制御: ロール/権限により閲覧制限（公開範囲を限定）。
- 一般開示（公開）：
  - 公式Web/ポータル、GitHub `docs/`、必要に応じてナレッジベースで公開。
  - パーマリンク/リリースノートで参照を固定。

### 招集・周知の標準手段（最低要件）
- 告知文に含める必須要素（本文が定める項目を包含）：開始日、期間、電磁的方法、該当決議レベル、議案概要、投票選択肢。
- 実施基準：限定開示チャネルで2系統以上（例: Snapshot＋Discord）。必要に応じてリマインドを1回以上。
- テンプレート参照：`review/04-focused/templates/` の告知テンプレートを使用。

### 記録・保存の最低要件（証跡/監査）
- 保存物：
  - 告知メッセージURL、提案/投票ページURL（パーマリンク）
  - スクリーンショット（PDF/画像）
  - GitHubへの記録（議事録/結果レポート、コミットIDで改ざん耐性確保）
- 運用：リリースタグ/固定リンクで参照を安定化。

### テンプレート/様式（配置の指針）
- 議事録・開示文面・告知文のテンプレートはリポジトリの `review/04-focused/templates/` などテンプレート用ディレクトリに配置する（該当時）。
- 名簿・フォームの様式は Google Form/Spreadsheet 上で管理し、参照リンクを当該テンプレート集に記載する。

備考: 具体的な数値や閾値などの可変値は `dao-assembly/DAO総会別紙.md` および `tokens/トークン別紙.md` に保持し、本別紙はツールの枠組みに限定する。通知や開示の手続・期限は各規程本文を参照する。
