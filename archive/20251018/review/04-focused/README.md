## 04-focused 作業README（総会・トークン・運営・定款［最小変更］）

本ディレクトリは、当面の対象を以下の規程類に絞った集中作業用ワークスペースです。

- DAO総会規程（v2.1）
- トークン規程（v2.2）
- 運営規程（v2.1）
- 定款 v2.0（最小変更のみ）

### ディレクトリ構成
- `materials/`: 02-modified の確定版／定款v2.0のベースラインを配置
- `backlog/`: 積み残し・検討タスクを管理（本文修正と別紙移植を分離）
- `templates/`: 指示書・提案書などのテンプレート（別紙テンプレート含む）
- `appendices/`: 別紙一式。
  - `共通運用.md` にツール一覧（ミニマム）を集約（通知・開示の手続・期限は各規程本文）
  - `dao-assembly/` に投票プラットフォーム仕様、期間・レベル、議案リスト、結果開示、リカバリー等
  - `tokens/` に交換レート、取得制限/KYC、メタデータ、管轄情報 等

### 進め方
1. `materials/` のベースラインから着手する規程を選定
2. `backlog/` の積み残しから該当タスクを切り出し、担当・期限を設定
3. 本文は抽象度維持、運用・パラメータは `appendices/` へ移植
4. 修正結果は `review/03-modifications/` の差分ドキュメント更新で可視化
5. 承認後は `review/02-modified/` を更新、`new/` との差分は自動生成スクリプトで更新

### スコープ外（当面）
- トレジャリー管理規程
- DAO憲章
- 運営規程向けの個別別紙（当面は不要）

必要に応じて後続フェーズで再開します。

### 合意事項
- 「別紙」化できるパラメータは本文から分離し `appendices/` で一元管理
- 共通運用.md にはツールのみ（ミニマム）を集約
- 総会規程の投票プラットフォーム仕様・議決レベル別議案リストは `dao-assembly/`
- 用語の統一（例: 「トークンホルダー」→「議決権保有者」）
- 定款は最小限の変更に限定（他規程との参照整合・用語統一 など）
