# DAO総会規程 v1.0 → v2.0 差分比較

## 📋 概要

| 項目 | v1.0 | v2.0 | 変更内容 |
|------|------|------|----------|
| **決議レベル** | 5段階（レベル1-5） | 3段階（重要・通常・委任） | 🔄 67%簡素化 |
| **組織体制** | 委員会制度 | 業務執行社員中心 | 🔄 責任明確化 |
| **参加者** | 社員権トークンのみ | 社員権+ガバナンス | ➕ 参加者拡大 |
| **技術仕様** | 曖昧な電磁的方法 | Snapshot等明記 | ➕ 実装可能化 |
| **条文数** | 15条 | 15条+技術補足 | 🔄 内容充実 |

---

## 🔄 主要変更点

### 1. 決議制度の大幅簡素化

#### v1.0：5段階決議制
```
├── レベル1-2：（規程に記載なし）
├── レベル3：臨時投票（定足数なし）
├── レベル4：通常投票（5%定足数）
└── レベル5：特別投票（20%定足数）
```

#### v2.0：3段階決議制
```
├── 重要決議（20%定足数+過半数賛成）
├── 通常決議（5%定足数+過半数賛成）
└── 委任決議（定足数なし+過半数賛成）
```

### 2. 組織体制の抜本的変更

| 項目 | v1.0 | v2.0 |
|------|------|------|
| **運営主体** | 委員会（委員会長） | 業務執行社員 |
| **意思決定** | レップホルダー会議 | 業務執行社員判断 |
| **投票管理** | 委員会 | 業務執行社員 |
| **議長役** | 委員会長 | 代表社員または業務執行社員互選 |

### 3. 参加者の拡大とガバナンス民主化

#### v1.0：限定的参加
- **対象**: 社員権トークンホルダーのみ
- **例外**: レベル4承認により非トークンホルダーの提案権付与

#### v2.0：包括的参加
- **対象**: トークンホルダー（社員権+ガバナンス）
- **権利**: 提案権・議決権を平等に付与

## 📊 技術仕様の具体化

### v1.0：曖昧な技術要件
```
投票方法：「電磁的方法」
具体的ツール：「本DAOがコミュニケーションツールとして利用する●●」
議事録：「技術的に変更が不可な形で」
```

### v2.0：明確な技術仕様
```
【技術仕様補足】
- 主要プラットフォーム: Snapshot
- 補完プラットフォーム: EVM互換の投票管理ツール
- 対応ウォレット: MetaMask等EVM互換ウォレット
- 議事録: ブロックチェーン技術による改ざん不可能な記録
```

---

## 🎯 決議事項の整理と明確化

### 重要決議の整理

#### v1.0：レベル5（特別投票）
- 社員権トークン取得資金の使途決定・承認
- 利益配当・残余財産分配
- 業務執行社員の選任・解任
- 定款・規程変更
- 解散・組織変更・吸収合併等

#### v2.0：重要決議
- 定款変更
- 業務執行社員・代表社員等の選任・解任
- 利益配当・残余財産分配
- 利益相反取引承認
- 支配人の選任・解任
- 社員退社・清算人解任
- 組織変更・吸収合併等

**→ 🔄 内容の明確化と体系化**

### 通常決議の拡充

#### v1.0：レベル4（通常投票）
- 利益相反取引承認
- 委員会長選任・解任
- 定款・規程変更（レベル5以外）
- レベル3事項の決定

#### v2.0：通常決議
- 各種規程の制定・改廃
- 予算承認
- 重要な業務執行方針
- 重要決議以外の重要事項

**→ ➕ 業務執行の具体化**

## 🛡️ ガバナンス制度の強化

### 社員同意手続きの明確化

#### v1.0
- 第5条で「免責事項」として拒否権を規定
- 具体的な社員同意手続きの記載なし

#### v2.0
- 第11条で社員同意手続きを独立して規定
- 定款第18条との明確な連携
- 社員権トークンホルダー全員の同意取得

### 拒否権制度の明確化

#### v1.0
- 第11条「免責事項」として曖昧に規定
- 業務執行社員の判断による拒否権

#### v2.0
- 第12条で拒否権を独立して規定
- 定款第19条との明確な連携
- 行使要件の明確化

---

## 📈 意思決定の効率化

### 投票期間の設定

#### v1.0：期間設定なし
- 各決議レベルの投票期間が不明確
- 緊急時の短縮手続きのみ規定

#### v2.0：段階別期間設定
- **重要決議**: 7日間以上
- **通常決議**: 5日間以上
- **委任決議**: 3日間以上

### 定足数要件の合理化

| 決議種別 | v1.0 | v2.0 | 効果 |
|----------|------|------|------|
| **最重要** | 20%定足数 | 20%定足数 | 同じ |
| **重要** | 5%定足数 | 5%定足数 | 同じ |
| **軽微** | 定足数なし | 定足数なし | 同じ |

**→ 🔄 実績のある要件を継承**

---

## 🔄 権限・責任の明確化

### 業務執行社員の権限強化

#### v1.0での業務執行社員
- 拒否権の行使
- 説明義務の履行
- 委員会との協働

#### v2.0での業務執行社員
- DAO総会投票の管理
- 議長の互選
- 委任事項の執行
- 説明義務の履行

**→ ➕ 明確な運営責任**

### 議案提案プロセスの体系化

#### v1.0：複雑な段階
1. 議題提案チャンネルでのスレッド作成
2. 5名以上の賛同取得
3. レップホルダー会議での重要度審議
4. ガバナンス事務局長による全体告知
5. 投票実施

#### v2.0：シンプルな段階
1. コミュニケーションプラットフォームでの提案
2. 5名以上の賛同取得
3. 業務執行社員による重要度審議
4. 全体告知
5. 投票実施

**→ 🔄 中間組織の削減**

## 📊 影響分析

### 🟢 改善点
1. **意思決定の迅速化**: 5段階→3段階による簡素化
2. **責任の明確化**: 委員会制→業務執行社員中心
3. **参加者の拡大**: ガバナンストークンホルダー参加
4. **技術実装性**: 具体的ツール（Snapshot）の明示
5. **法的整合性**: 定款との完全整合

### 🟡 検討点
1. **権限集中リスク**: 業務執行社員への権限集中
2. **移行コスト**: 既存システムからの移行負荷
3. **技術依存**: Snapshot等特定ツールへの依存
4. **運用習熟**: 新制度への適応期間

### 🔴 リスク要因
1. **業務執行社員の負荷**: 管理業務の集中
2. **技術トラブル**: プラットフォーム障害時の対応
3. **参加者教育**: 新制度の理解・普及
4. **法的変更**: 会社法改正等への対応
