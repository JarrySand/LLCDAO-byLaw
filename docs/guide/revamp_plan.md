# ガイド刷新計画 (Guide Revamp Plan)

## 1. 目的とゴール
**「このガイドを見れば、定款と規程のひな形を使って誰でもDAOを立ち上げ・運用できる」** 状態を目指します。
最終成果物は **GitHub Pages** として公開し、ブラウザで閲覧しやすいオンラインドキュメント（Wiki形式など）として整備します。

## 2. ターゲット読者
- **DAO立ち上げ希望者**: 法的知識や技術知識が必ずしも深くないが、コミュニティやプロジェクトのためにDAOを作りたい人。
- **実務担当者**: 実際に規程をカスタマイズしたり、日々の投票管理を行うオペレーター。

## 3. ガイドの全体構成（案）
大きく「立ち上げ」と「運用」の2部構成とし、既存の資料（`context`, `structure`, `dao-setup`, `dao-runbook`）をこの体系の中に統合します。

### 導入 (Introduction)
- **はじめに**: このひな形の思想とできること（`context.md` の内容など）
- **全体像**: 定款・規程・別紙の関係図（`structure.md` の内容）

### Part 1: DAO立ち上げガイド (Setup Guide)
**前提条件（責任分界点）**:
このガイドは、DAOの企画・デザイン（トークンエコノミクス、コミュニティの目的、事業内容など）が**最低限固まっている状態**を前提とします。「どんなDAOにするか」のアイデア出しではなく、「決まったデザインをどう法的・実務的な枠組み（定款・規程）に落とし込むか」に焦点を当てます。

1. **基本設計（Concept）**
   - あなたのコミュニティはDAOに向いているか？（スコープ確認）
   - どの種類の規程セットを使うか
2. **パラメータ設計（Parameter）**
   - 定款・規程スキャンに基づく必須項目の入力ガイド
   - 「別紙」の具体的な埋め方
3. **法人設立準備（Legal）**
   - 定款の確定プロセス
   - 登記手続きの概要
4. **ツール導入（Tech）**
   - Discord / コミュニケーションツール設定
   - 投票ツール（Snapshot等）の設定
   - ウォレット設定

### Part 2: DAO運用ガイド (Operation Guide)
立ち上げ後の日常運営、特に「提案→決議→執行」のサイクルを回すためのマニュアル。
*（既存の `dao-runbook/` をベースに、規程全体をスキャンして網羅性を確保する）*

1. **意思決定サイクル（Governance）**
   - 提案（Proposal）の書き方テンプレート
   - 議論・審議の進め方
   - 投票の実施手順
   - 決議結果の確定と執行（議事録作成含む）
2. **メンバー管理（Membership）**
   - 新規メンバー加入フロー（トークン付与・名簿登録）
   - メンバー脱退フロー
3. **定期的メンテナンス（Maintenance）**
   - 定款・規程のアップデート手順
   - 別紙（パラメータ）の変更手順
4. **その他定常業務**
   - 規程スキャンにより洗い出した運営タスク（報告、公開、監査対応など）

### 付録 (Appendix)
- 用語集
- トラブルシューティング
