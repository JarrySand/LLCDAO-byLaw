# DAO立ち上げガイド（Setup Guide）

このガイドは、雛形（定款/各規程/別紙）に「従って」DAOを新規に立ち上げる際に、何を決め・準備し・実行すべきかを網羅的に示します。各章は TL;DR（5分）とチェックリスト中心で、決定すべきパラメータを明示します。

## Start Here（フェーズ別）
- **Phase 1: 目的・範囲・法域の確定**
- **Phase 2: パラメータ設計（規程・別紙のカスタマイズ）**
- **Phase 3: 法人設立・規程採択**
- **Phase 4: ツール導入・ローンチ**

## 1. 目的・範囲・法域の確定（ガバナンス方針）
まず、この合同会社型DAOモデルがあなたのコミュニティに適しているか確認します。
詳細は `docs/guide/context.md` も参照してください。

- [ ] **目的**: DAOで何を達成するか（定款 第2条「目的」に記載する事業内容は？）
- [ ] **法域**: 日本国内の合同会社として登記することを前提としているか
- [ ] **社員構成**: 少数の業務執行社員 + 多数のDAOメンバー（社員権トークン保有者）という構造に合意しているか

## 2. パラメータ設計（規程・別紙のカスタマイズ）
スキャン結果に基づき、以下の項目を決定してください。

### 定款 (Articles of Incorporation)
| 項目 | 決定事項 |
| :--- | :--- |
| **商号** | `●●DAO合同会社` (英文表記も決定) |
| **本店所在地** | 登記する住所（バーチャルオフィス等も可だが要確認） |
| **公告方法** | 官報掲載か、電子公告か |
| **代表社員** | 誰が代表社員になるか（実名登記が必要） |
| **事業年度** | 何月何日から何月何日までか |
| **資本金** | 設立時の資本金額 |

### 別紙 (Appendices) - ガバナンスの肝
`appendices/` フォルダ内の各ファイルを直接編集します。

**A. DAO総会別紙 (`DAO総会別紙.md`)**
- [ ] **投票方式**: 単純多数決にするか、重み付きにするか
- [ ] **重要決議**: 投票期間（例: 7日）、定足数（例: 20%）、可決ライン（例: 過半数）
- [ ] **通常決議**: 投票期間（例: 5日）、定足数（例: 5%）、可決ライン（例: 過半数）

**B. トークン別紙 (`トークン別紙.md`)**
- [ ] **チェーン**: メインで使用するブロックチェーン（ChainID）
- [ ] **トークン設計**:
    - **社員権**: 譲渡制限の有無、発行上限
    - **ガバナンス**: 譲渡可能か、配布ルール
    - **リワード**: ユーティリティ、交換レート
- [ ] **KYC/制限**: 大量保有時の制限ルール、KYCの要否

**C. 共通運用 (`共通運用.md`)**
- [ ] **コミュニケーション**: Discord, Telegram 等の指定
- [ ] **投票ツール**: Snapshot, On-chain Gov 等の指定
- [ ] **ウォレット**: トレジャリー管理用マルチシグ（Gnosis Safe等）の指定

## 3. 法人設立・規程採択
決定したパラメータを元に、実際のドキュメントを作成・確定させます。

1. **定款の確定**:
   - 定款ファイル（`01_`で始まるファイル）の `●●` 部分をすべて置換し、Word等でPDF化して公証役場/法務局へ提出可能な形式にする。
2. **登記申請**:
   - 司法書士へ依頼するか、自身で登記申請を行う。
3. **規程の採択**:
   - 設立時社員による同意（または設立後の社員総会）にて、`02`〜`04`の規程および別紙を正式に採択する。
   - 採択日、施行日を各ファイルの冒頭/末尾に記載する。

## 4. ツール導入・ローンチ
物理的な準備と、最初のメンバー受け入れを行います。

- [ ] **ツールセットアップ**:
    - Discordサーバー構築（権限設定）
    - Snapshotスペース作成（投票設定）
    - Gnosis Safe作成（署名者設定）
- [ ] **トークン発行**:
    - コントラクトのデプロイ（または発行ツール使用）
    - 初期メンバーへの配布
- [ ] **ローンチ**:
    - 設立アナウンス
    - 第1回DAO総会（キックオフ）の開催
