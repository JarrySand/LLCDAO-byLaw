# 2. パラメータ設計（規程・別紙のカスタマイズ）

## 「空欄を埋める」作業
ここでは、具体的な数値や設定を決めます。これらは「別紙」と呼ばれるファイルに書き込むことで、規程本文を修正せずにルールを設定できます。

### A. 定款の空欄（基本情報）
ルートディレクトリにある定款ファイル（`01_`で始まるファイル）の `●●` 部分を埋めるための情報です。

| 項目 | 意味・決め方のヒント | 決定値（メモ） |
| :--- | :--- | :--- |
| **商号** | 会社の名前。「●●DAO合同会社」など。英語表記も決めましょう。<br>**Point**: 既存の会社と名前が被っていないか、国税庁の法人番号公表サイト等で検索して確認しましょう。 | ____________________ |
| **目的** | 定款第2条。可能な限り広く（将来の拡張性）かつ、何をする会社か分かる具体性（銀行審査対策）のバランスをとって箇条書きします。 | (箇条書き) |
| **本店所在地** | 登記する住所。自宅、オフィス、バーチャルオフィス等。<br>**Point**: 賃貸物件を本店にする場合、大家さんの許可が必要です。バーチャルオフィスは便利ですが、銀行口座開設の難易度が上がることがあるので、事前に銀行に相談するのも手です。 | ____________________ |
| **公告方法** | **Point**: 「官報」が年間約7.5万円（決算公告のみ）〜で手間が少ないです。「電子公告」は登記簿にURLを載せる必要があり、サイト維持義務が発生しますが、公告掲載費自体はかかりません。初期は官報が無難です。 | 官報 / 電子公告 |
| **代表社員** | 誰が代表になるか。<br>**Point**: 業務執行社員（役員）全員が代表権を持つのが原則ですが、混乱を避けるため定款で「代表社員」を1名に絞る（互選規定を入れる）のが一般的です。 | ____________________ |
| **事業年度** | 決算の区切り。<br>**Point**: 「消費税の免税期間（最大2年）」をフル活用するために、設立日の前月末を決算日に設定する（第1期を約1年にする）テクニックがよく使われます。 | ____月____日 〜 |
| **資本金** | 設立時にいくら入れるか。<br>**Point**: 1円でも作れますが、法人口座開設や取引先の信用を考えると、最低でも数万〜数十万円は入れるのが望ましいです。資本金1,000万円未満なら設立時の消費税が免税になります。 | ____________________円 |

### B. ガバナンス設定（DAO総会別紙）
`appendices/DAO総会別紙.md` を編集して設定します。

- **重要決議（重い決定）**
    - 定款変更や解散など、DAOの根幹に関わること。
    - **投票期間**: `7日` 以上推奨。<br>**Point**: あまり短いと「気づかなかった」という不満が出ます。土日を挟める期間にしましょう。
    - **定足数**: `20%` 程度。<br>**Point**: これより高いと、アクティブ率が下がった時に何も決められなくなります。
    - **可決ライン**: `過半数`。

- **通常決議（日常の決定）**
    - ルール変更や予算承認など。
    - **投票期間**: `3日〜5日`。<br>**Point**: 議論はフォーラムで十分に行い、投票期間は確認程度にする運用もアリです。
    - **定足数**: `5%〜10%`。<br>**Point**: 最初は低めに設定し、参加者が増えてきたら上げる（攻撃耐性を高める）のが安全です。

### C. トークン設定（トークン別紙）
`appendices/トークン別紙.md` を編集して設定します。

- **使用するチェーン**: Ethereum, Polygon, Base, Astar 等。<br>**Point**: メンバーが使い慣れているチェーンか、ガス代が安いL2を選びましょう。Snapshot投票だけならガス代はかかりません。
- **社員権トークン**:
    - 譲渡制限: 原則「あり」。<br>**Point**: これがないと、知らない人が勝手に社員（責任者）になってしまうリスクがあります。
