# 1. 目的・範囲・法域の確定

## 「何のために、どこで」を決める
このフェーズでは、まだ細かいルールを決める必要はありません。一番大事なのは**「この合同会社型DAOという仕組みが、自分たちのやりたいことに合っているか？」**を確認することです。

### チェックリスト
- [ ] **目的の明確化**: DAOで何を達成したいですか？
    - 定款第2条の「目的」に記載する事業内容を箇条書きにします。
    - **ポイント**: 
        - 可能な限り「広く」書いておくことで、ピボット（方向転換）時に定款変更の手間を省けます（例: 「その他付帯する一切の事業」を入れる）。
        - 一方で、「何でも屋」に見えないよう、メインの活動は具体的に書き、DAOのアイデンティティを明確にするバランスが重要です。
        - 銀行口座開設時に「事業内容が不明瞭」として審査に落ちないよう、誰が見ても分かる表現を心がけましょう。
- [ ] **法域の確認**: 日本国内の「合同会社」として登記しますか？
    - このひな形は**日本法**に準拠しています。
    - メンバーが匿名でも参加できますが、**代表者（業務執行社員）は実名で登記**され、法的責任を負う必要があります。
- [ ] **メンバー構造**: 「少数の責任者 + 多数の参加者」でOKですか？
    - **業務執行社員**: 実名。法的責任を負う。拒否権を持つ。
    - **DAOメンバー（社員権トークン保有者）**: 匿名OK。出資額以上の責任を負わない（有限責任）。投票で意思決定に参加する。
    - ※完全なフラット型（全員が同等の法的責任）を目指す場合、このモデルは不向きかもしれません。
