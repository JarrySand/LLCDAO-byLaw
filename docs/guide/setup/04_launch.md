# 4. ツール導入・ローンチ

## デジタル空間の準備
法人ができたら、次はデジタルの「場所」を作ります。

### 1. コミュニケーションツール (Discord等)
メンバーが集まり、議論する場所です。
- **サーバー作成**: 新しいサーバーを立てます。
- **チャンネル設定**:
    - `#一般`: 雑談用
    - `#アナウンス`: 運営からの重要なお知らせ（**Point**: 全員が見る場所なので、書込権限はAdminのみにするのが鉄則です）
    - `#議題提案`: 誰でも提案スレッドを立てられる場所
- **権限設定 (Role)**:
    - `Member`: 社員権トークンを持っている人（**Point**: Collab.LandやGuild.xyzなどのBotを使うと、トークン保有者に自動でロールを付与できて管理が楽です）
    - `Admin`: 運営担当者

### 2. 投票ツール (Snapshot等)
ブロックチェーンを使わない（オフチェーン）投票なら Snapshot が標準的です。ガス代がかかりません。
- **Space作成**: [Snapshot.org](https://snapshot.org) でスペースを作ります（ENSドメインが必要な場合があります）。
- **戦略設定**: 「ERC-721（NFT）を1個持っていたら1票」という設定をします。
- **Point**: "Voting Period"（投票期間）のデフォルト値を、規程で決めた「通常決議」の日数（例: 5日）に合わせておくとミスが減ります。

### 3. ウォレット (Gnosis Safe)
DAOのお財布（トレジャリー）です。代表者個人の財布ではなく、複数人の鍵が必要な「マルチシグ」を使います。
- **Safe作成**: [Safe{Wallet}](https://app.safe.global/) で作成。
- **署名者設定**: 業務執行社員のアドレスを登録します。
- **閾値設定**: 「3人中2人が承認したら送金できる」など設定します。
- **Point**: 閾値が「1/N」だと誰か1人がハッキングされたら資産が盗まれます。必ず「2以上」に設定しましょう。

### 4. トークン発行と配布
社員の証となる「社員権トークン（NFT）」を発行します。
- **発行**: 第三者ツール（Manifold, Thirdweb等）を使えば、コードを書かずに発行できます。
- **配布**:
    - 設立時社員のウォレットに1つずつ送ります。
    - 新しいメンバーが入ってきたら、その都度送る（または購入してもらう）形になります。
- **Point**: 将来的にOpenSeaなどで勝手に売買されないよう、SBT（譲渡不可）にするか、コントラクトで`transfer`機能を制限できるか確認しましょう。

### 5. ローンチ（DAO総会 第1回）
準備が整ったら、メンバーを集めて最初のDAO総会を開きましょう！
- **議題**: 「設立の報告」「初期メンバーの顔合わせ」「最初の予算承認」など。
- **投票テスト**: 実際にSnapshotで投票してもらい、システムが動くか確認します。

**おめでとうございます！これでDAOの立ち上げは完了です。**
明日からは [運用ガイド](../operation/README.md) を見ながら、日々の運営を行ってください。
