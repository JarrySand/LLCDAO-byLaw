## 4. 提案（議案）ライフサイクル運用

TL;DR（5分で実行）
- テンプレに沿って提案を提出（必須項目を網羅）
- 週次トリアージで状態/担当/期限を更新
- 採決スケジュールと方式を掲示（検証可能性を確保）

実行メタデータ
- 担当: 運営責任者/提案者/記録
- 所要: 30–90分（提案規模による）
- 頻度: 随時/週次
- 依存: 02 カレンダー、03 総会、06 トレジャリー

目的: 良い提案が確実に届き、適切に審査・可視化・実行される流れを確立する。

### 状態遷移
Draft → Submitted → Review → Voting → Passed/Rejected → Implementing → Completed

### 実務手順
- 受付: テンプレで必須項目（背景/目的/影響/代替案/コスト/リスク/実行計画）を網羅
- 審査: 適格性・重複・依存関係・予算/権限の整合
- 採決: 方式・閾値・定足確認、検証可能な記録
- 実行: 責任者・期限・KPI・リスク対応、進捗公表
- 完了: 成果/差分/学びの記録、継続施策の提案

### 可視化ダッシュボード（最小要件）

---

前へ: 03 総会運用 / 次へ: 5 トークン・権限
- 提案一覧（状態/担当/期限/予算）
- 投票予定と結果
- 実行中タスクとリスク
