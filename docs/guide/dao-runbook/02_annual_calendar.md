## 2. 年間運用カレンダーと定常業務

TL;DR（5分で実行）
- 来月分の締切・会合日・採決ウィンドウを1枚に確定
- ダッシュボード（提案/投票/資金）を更新して共有
- 週次トリアージの時間枠をチームに固定化

実行メタデータ
- 担当: 運営責任者
- 所要: 45分（初回90分）
- 頻度: 週次/月次/四半期
- 依存: 01 役割、03 総会、04 提案

目的: 予見可能なスケジュール運用で準備不足を回避し、合意形成コストを低減する。

### 典型カレンダー（例）
- 毎月: 提案受付締切、レビュー会、定例報告、資金執行ウィンドウ
- 四半期: 監査/棚卸、KPIレビュー、パラメータ見直し
- 半期: 戦略レビュー、役割ローテーション
- 年次: 事業計画/予算、重要改訂の総会

### 定常業務チェックリスト
- [ ] 提案受付→整形→掲示を毎週定時に実施
- [ ] アクティブ提案の状態（審査/投票/実行）を更新・通知
- [ ] トレジャリー残高/コミットメントを可視化
- [ ] トークン供給/ロック/権限ログを更新

### 運用プロトコル（例）

---

前へ: 01 役割と責務 / 次へ: 03 DAO総会の準備・開催・記録
- 週次30分: 提案トリアージとアジェンダ更新
- 月次60分: 財務・リスクレビュー、規程変更の必要性評価
- 四半期90分: 監査・KPI・パラメータ総点検
