## 定款・規程の設計思想（Context）

本ドキュメントは、今回の定款・各規程の「共通の設計思想・前提・意図」を記述します。個人の信条ではなく、ひな形として共有される原則を対象とします。

### 1. 問題設定とゴール（この雛形が解く課題）

日本国内において「DAO」の概念と実践を実社会の課題解決や市場創出に活用する潮流が生まれつつあります。web3ネイティブな世界から拡張して実社会においてDAOを活用するには、いくつもの課題があります。

- 契約主体
- 法定通貨、銀行口座の取り扱い
- 有限責任の適用可否 など

これらの課題の解決策として期待されるのが法人制度の活用であり、日本においても2024年4月の内閣府令改正により、その実現に向けた最初の一歩を踏み出しました。誰も明確な答えを持たない実社会での法人を活用したDAOの実装には、社会全体での試行回数の積み重ねが必要であり、その試行のコストを下げることが重要な課題です。

web3のオープンソース文化の中で、拡張性があり複製可能な様々な技術的なツールの開発は進められています。一方で、「日本国内法に対応する法務支援ツール」は未整備です。各DAOがそれぞれで独自の定款や規程、契約書等を作成しようとした場合、数十から数百万円規模の予算が必要となり、これは試行を重ねるうえでの大きな障害となります。

本プロジェクトは、DAOの法人利用時の定款・規程の汎用的な雛形を提供することにより、次の実現を目指します。

- 「試行における各DAOの法務コスト（限界費用）」を限りなくゼロに近づける
- 適法性の高い運用によりDAOの社会受容を促進する

DAOの社会実装を推進していきます。その最初期の実装として、最も自由度が高く運営コストが小さい合同会社を対象としています。

本プロジェクトは、DAOの社会実装を後押しする公共財（Public Good）として位置づけます。成果物はオープンソースとして公開し、実践者・研究者・専門家を含むコミュニティによる検証、改善提案、再利用・派生（フォーク）を歓迎します。

### 2. スコープ

本雛形が適用できるDAO／適用が難しいDAOを明示します。

- 適用できるDAO（対象）
  - 日本の合同会社を活用するDAO
  - 社員権トークンの取得（販売）により、参加者が社員として加入する設計
  - オープン参加と情報公開を重視し、オンチェーン/オフチェーン投票を組み合わせるガバナンス

- 適用が難しいDAO（非対象）
  - 完全匿名を前提とする組織（社員名簿が必要なため）
  - 匿名メンバーの合意が重要事項の最終決定となる、完全分散型の意思決定を求める設計
    - 法的な責任所在の明確化のため、社員の同意（最終承認）や業務執行社員の拒否権が必要で、DAO決議のみで完結しない議案がある
  - 合同会社以外の法人形態を前提とする場合（一般社団、株式会社 等）
  - 社員権トークンを用いない、または社員権を販売しない設計
  - 少人数のクローズドな投資クラブ等、公開や参加拡張を想定しないDAO

- 制約の理由
  - 合同会社の制度（法律の前提）
    - 社員の身元情報を社員名簿に記載する必要がある → 完全匿名は不可
    - 会社法で社員の同意が必要な議決は社員の同意等の手続が必要 → DAO決議だけで完結しない議案がある（本雛形は「DAO決議→社員同意」で運用）
    - 代表者/業務執行の選任と責任の所在が必要 → オンチェーン合意のみでの自動執行は不可、オフチェーン手続と接続

### 3. 設計原則（例: 透明性、最小権限、可監査性、操作可能性）

- 多元性  
  DAOにおけるプロトコルやガバナンスを制限せず、様々な思想やシステムを実現する土台となるよう、各ドキュメントのモジュール化を行っています。  
  例：雛形として提供している「DAO総会規程」のプロトコルは、一般的にDAOで使われやすい参加型予算（Participatory budgeting。参加者から資金を集め、参加者の提案を参加者の投票で採択してリソースを分配する）形式ですが、他のプロトコルを採用したい場合はこのDAO総会規程の意思決定構造を変更するだけで、その他の規程類の調整は最小限で済むように設計しています。

- 透明性  
  ブロックチェーンの透明性に倣い、各種ルールや規則、情報を可能な限り公開することを原則にしています。これにより、以下を目指しています。
  - DAOに参加する人は自ら情報を把握して納得したうえで参加できる
  - すべての人による監視体制によって不正を予防する

- 操作可能性  
  法務に詳しくない方でも自身が実現したいプロトコルを作成しやすいよう、各パラメータ（投票力やトークンと交換できるものなど）を別紙にまとめることで創意工夫を行いやすくしています。

- 責任所在の明確化  
  日本にはDAO法が存在しないため、一般の法人と同じように法的な責任が適用されます。つまり、部分的な顕名化や法的責任の追及が必要となる場合があります。本定款・規程では、実情として誰がどのように責任を負うのか、逆に免責されるのはどのような場合なのかを明示することを目指しています。

- 内/外要因によるDAO破綻時の法人としての対応の明確化（Business Cancellation Policy）  
  例えば、根幹となるブロックチェーンの不具合発生時の対応など、意思決定機関であるDAOが破綻した場合の法人としての対応の明確化を目指しています。

- DAOによる意思決定の影響力の最大化  
  現行法で可能な範囲において、DAOによる意思決定が法人の意思決定に直結する仕組みづくりと、逆に法的責任からそれが難しい範囲を明確に線引きする制度設計を行っています。

### 4. トレードオフと選択理由（何を捨て、何を採ったか）

- 完全自動の分散執行 vs 法的確実性・責任  
  採用: 法的確実性（重要事項は「DAO投票→社員の同意」で確定、業務執行社員の拒否権を保持）  
  説明: 現行法では業務執行の責任をDAO構造で免除できないため、責任を負う人を適法に特定できるよう、定款変更・代表者選任・資金の大口支出等は二段階で確定し、違法/不当決議の実行を抑止  
  調整先: `02_DAO総会規程`、`03_運営規程`

- 参加の開放性（トークン加入） vs 顕名責任（社員名簿）  
  採用: 開放性＋最小限の顕名（名簿はアクセス制御）  
  説明: 現行法で社員名簿による当事者特定が必要なため、参加は社員権トークンで広げつつ、名簿は必要最小限の顕名とアクセス制御で運用  
  調整先: `01_定款`、`appendices/共通運用.md`

- 定款の安定性（枠組み固定） vs 運用の柔軟性（可変値の迅速変更）  
  採用: 定款＝枠組み、可変＝規程/別紙（投票期間・定足数・ツール等）  
  説明: 定款変更は重いため、投票期間・定足数・使用ツール等の可変項目は規程/別紙で管理し、迅速な更新と将来のツール入替に対応  
  調整先: `appendices/DAO総会別紙`、`appendices/トークン別紙`、`appendices/共通運用.md`

- 規制適合の確実性 vs 強い金銭的権利設計  
  採用: 規制適合を優先（ユーティリティ中心、配当等は明確に制御）  
  説明: 金銭的配当等は規制影響が大きいため、ユーティリティ中心に設計し、配当等の扱いは明確に制御してコンプライアンスと再現性を確保  
  調整先: `04_トークン規程`、`appendices/トークン別紙`

- プロトコル中立性 vs 導入容易性  
  採用: 既定＝参加型予算、モジュール交換で差し替え可能  
  説明: 初期は分かりやすい参加型予算を採用しつつ、意思決定構造をモジュールとして差し替え可能にして多様なプロトコルを許容  
  調整先: `02_DAO総会規程`（意思決定構造の差し替え）

- 透明性（公開・監査可能性） vs プライバシー配慮  
  採用: 原則公開＋名簿等は必要最小限の顕名・アクセス制御  
  説明: 規程・決議・運用情報は原則公開し監査可能性を確保、個人情報を含む名簿等は限定公開として参加者の安全を守る  
  調整先: `03_運営規程`、`appendices/共通運用.md`

- 迅速性（短いレイテンシ） vs 監査可能性（適切な審議・記録）  
  採用: 最低限の審議・公開・記録プロセスを確保  
  説明: 投票前の最小限の審議・公開・記録を必須化し、誤実行を防ぎつつ意思決定の正当性を担保（投票期間・定足数で調整）  
  調整先: `appendices/DAO総会別紙`（投票期間・定足数）
