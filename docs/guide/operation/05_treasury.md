# 5. トレジャリー運用（資金管理）

DAOのお金（トレジャリー）は、みんなの共有財産です。透明性と安全性を最優先します。

## 基本ルール
- **マルチシグ必須**: Gnosis Safe等のマルチシグウォレットを使い、必ず「複数人の承認」がないと送金できないようにします。
- **個人管理禁止**: 代表者個人の口座やウォレットでDAOの資金を混同して管理してはいけません。

## 出金フロー（Spending）

### 1. 予算承認（事前）
原則として、DAO総会で承認された「予算」の範囲内でしかお金は使えません。業務執行社員の一存で勝手にお金を使うことは「横領」や「背任」になりかねないため、必ず事前の承認を得ます。

- **少額経費（包括承認）**: 
    - 毎回の投票は手間なので、「月額〇〇万円までの雑費（サーバー代、ドメイン代など）」としてまとめて予算承認しておくと便利です。
    - これにより、日々の細かい支払いは業務執行社員の判断でスピーディーに行えるようになります。
- **多額の出費（個別承認）**: 
    - 新しいプロジェクトや高額なツールの導入などは、個別に提案書を作成し、投票で承認を得ます。
    - 「何に、いくら、なぜ必要なのか」をメンバーに説明し、納得してもらうプロセスです。

### 2. トランザクション作成
支払いの必要が生じたら、業務執行社員の1人がSafeでトランザクション（送金データ）を作成します。
- **宛先**: 間違いがないかトリプルチェック。コピペミスによる誤送金（いわゆるGOX：資産の消失）は二度と戻ってきません。
- **金額**: 承認された予算内であるか確認します。

### 3. 署名と実行
他の業務執行社員（署名権者）に連絡し、承認（署名）を依頼します。
- 設定された閾値（例: 2/3）の署名が集まると、実際にブロックチェーン上で送金が実行されます。
- これにより、「誰か1人が勝手に資金を持ち逃げする」リスクを防ぎます。

## 記帳と報告（Accounting）
お金を使ったら、必ず「記録」と「報告」をします。これがDAOの信頼性の根幹です。

- **記録（帳簿）**: 
    - いつ、誰に、何のために、いくら払ったかを表計算ソフト等で記録します。
    - これが将来の「決算書」の元データになります。
- **証憑（しょうひょう）の保存**: 
    - 請求書や領収書のデータは、Googleドライブ等の共有フォルダに保存します。
    - 税務調査が入った際に「確かに事業のために使った」と証明するために必須です。

- **月次報告**: 月に一度、入出金の明細をメンバーに報告します。ブロックチェーンのエクスプローラー（Etherscan等）のリンクを貼ると透明性が高まります。
