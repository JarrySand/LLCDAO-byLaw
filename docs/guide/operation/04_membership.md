# 4. メンバー管理とトークン

「誰がDAOのメンバーか」は、トークン（NFT）が証明します。

## 新規メンバーの加入（Onboarding）

### 手順
1. **入会希望**: Discordやフォームで申請を受け付けます。
2. **審査/承認**:
    - **オープン型**: 誰でもウェルカムなDAOなら、申請があれば即承認します。
    - **許可制**: スキルや貢献意欲を見る場合、面談や既存メンバーによる投票を経て承認します。
3. **トークン送付**:
    - 運営がウォレットから、新メンバーのアドレスへ「社員権トークン」を送ります。
    - **Point**: ブロックチェーンアドレスは1文字でも間違えると届きません。最初は少額のテスト送金をするか、コピペしたアドレスを「最初と最後の4文字」で目視確認する癖をつけましょう。
4. **名簿更新（最重要）**:
    - 合同会社では「社員名簿」の整備が法律で義務付けられています。
    - 定款別紙の「社員名簿」（Googleスプレッドシート等で管理）に、新しい行を追加します。
    - **必須項目**: 氏名（または法人名）、住所、出資額（トークン購入額など）、ウォレットアドレス。
    - ※匿名DAOの場合でも、運営（業務執行社員）だけは法的に本人確認情報を把握しておく必要があります。
5. **ロール付与**:
    - Discordで `Member` ロールを付与します。Collab.LandやGuild.xyzなどのBotを使えば、「トークンを持っている人」に自動でロールがついたり外れたりするので、管理が非常に楽になります。

## メンバーの脱退（Offboarding）

### 手順
1. **退会届**: 本人から「辞めたい」という申し出を受けます。または、連絡不通や規約違反による「除名」の場合もあります。
2. **トークン回収/無効化**:
    - **返送**: 可能なら、トークンを運営用ウォレットに送り返してもらいます。
    - **バーン**: 送り返す代わりに、`0x0000...dead` という「誰も使えないアドレス（バーンアドレス）」に送ってもらう方法もあります。
    - **強制無効化**: SBT（譲渡不可トークン）や連絡がつかない場合、コントラクトの管理者権限で `Revoke`（剥奪）するか、そのTokenIDを「投票権なしリスト（ブラックリスト）」に登録して、実質的に無効化します。
3. **名簿更新**: 
    - 社員名簿からその行を削除するか、「退社日」を記入してグレーアウトします。
    - 誰がいつ辞めたかの記録は、将来のトラブル防止のために残しておくことを推奨します。

## トークンの管理（Security）
- **秘密鍵の管理**: 
    - 運営用ウォレットの秘密鍵（またはシードフレーズ）は、金庫の鍵と同じです。絶対にチャットに貼ったり、クラウドのメモ帳に保存したりしないでください。
    - 万が一漏れると、DAOの資金が全額盗まれる可能性があります。
- **紛失対応**: 
    - メンバーが「PCが壊れてウォレットに入れなくなった！」と言ってくることはよくあります。
    - **対応フロー**:
        1. **本人確認**: DiscordのDMやメールなどで、間違いなく本人であることを確認します。
        2. **旧トークンの無効化**: 無くしたウォレットに入っているトークンが悪用されないよう、ブラックリストに登録するなどの措置をとります。
        3. **再発行**: 新しいウォレットアドレスを聞き、そこへトークンを再発行（再送付）します。
    - **参照**: トークン規程 第13条に再発行の手順が定められています。
