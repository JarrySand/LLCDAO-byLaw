## 3. ガバナンストークン/社員権トークン設計と権限モデル

TL;DR（5分で決める）
- トークン種別（社員権/ガバナンス）の有無と関係
- 供給政策/ロック/譲渡制限の有無
- 権限（提案/投票/執行/委任）の付与基準

決定パラメータ
- 供給量・配布ポリシー・発行者
- 委任/代理・スラッシング/罰則の有無
- KYC/本人確認の基準

チェックリスト
- [ ] 別紙パラメータの初期値が定義
- [ ] 発行・回収・権限制御の手段が用意
- [ ] 監査手順（棚卸/ログ）を準備

---

サンプル初期設定（例）
- 種別: 社員権トークン=SFT（譲渡制限あり）、ガバナンストークン=ERC-20（委任可）
- 供給: 社員権=人数連動/非譲渡、ガバ=上限1億・配布比率 初期50%トレジャリー
- 権限: 提案=一定保有/ロール、投票=ガバ、執行=マルチシグ

- 例外時（不正/紛失）の回収手順と承認要件を明文化
