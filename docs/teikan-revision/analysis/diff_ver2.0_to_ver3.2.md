# 定款 ver2.0 → ver3.2 変更差分

**作成日**: 2025/12/08  
**目的**: 日本DAO協会ひな形（ver2.0）からver3.2への変更内容を関係者向けに説明

---

## エグゼクティブサマリー

### 変更の背景

ver2.0は日本DAO協会が作成したひな形であり、規程類が存在することを前提とした設計になっています。しかし、設立初期などで規程が存在しない場合の運用が困難でした。

ver3.2では、以下の設計思想に基づいて改定を行いました：

1. **規程がない場合**：定款のミニマムルールでスタンドアローンに機能
2. **規程がある場合**：ver2.0と同様にDAO総会規程がすべてをデザイン

### 主要な変更点

| 分類 | 変更内容 |
|------|----------|
| **ガバナンス構造** | 「社員同意・拒否権」方式 → 「業務執行社員会」方式 |
| **機能不全条項** | 4類型 → 1類型に限定 |
| **運営規程の権限** | DAO総会が制定・変更可能に |
| **規程変更の保護** | 業務執行社員会の「重要な事項」から規程の制定・変更を除外 |
| **フォールバック規定** | 各所に追加（規程なし時の対応） |

## 2. 削除された機能

### 2.1 社員同意プロセス（旧第18条）

**ver2.0の規定**:
```
会社法その他の法令により、社員による同意又は決定等が定められている事項については、
当該事項に関するDAO総会の議決の後にその内容に基づいて社員による書面または電磁的方法
による同意の取得を図るものとし、総社員の過半数の同意をもって決議するものとする。
```

**削除理由**: 業務執行社員会がブレーキ役を担うことで、社員同意プロセスは不要となった。

### 2.2 否認決議制度（旧第18条第3項）

**ver2.0の規定**:
```
○○日以内に同意しない場合は、DAO総会の決議の告知から○○日以内に社員の過半数による
賛同によるDAO総会議決に対する否認決議を行うことができる。
```

**削除理由**: 同上。業務執行社員会との両機関決議がセーフガードとして機能。

### 2.3 拒否権制度（旧第19条）→ 形式変更

**ver2.0の規定**:
```
DAO総会による議決が、次に定める免責事項に該当する場合に限り、業務執行社員の過半数に
よる議決をもって、DAO総会の議決の差し止めを行うことができる。
```

**変更内容**: 
- 削除ではなく、**規程存在時のみ有効な拒否権**（第18条）として再編
- 規程なし時は業務執行社員会との両機関決議がブレーキ役を担う

### 2.4 機能不全条項（4類型 → 1類型）

**ver2.0の規定（第25条）**:
```
3) 以下の条件のいずれかを満たし、当会社が機能不全に陥った場合
   1、DAO総会が開催されない期間が2年以上継続した場合
   2、DAO総会の決議に対する否認決議が10回連続かつ2年以上継続して行われた場合
   3、DAO総会の決議に対する拒否権の発動が5回連続かつ1年以上継続して行われた場合
   4、DAOの業務執行が一切行われない状態が6ヶ月以上継続した場合
```

**ver3.2の規定（第27条）**:
```
当会社の業務執行が一切行われない状態が2年以上継続した場合
```

**変更理由**: 
- 否認決議・拒否権を削除したため、関連する機能不全条項も削除
- 業務執行停止のみを2年に延長し、シンプル化

### 2.5 暗号資産非該当明記

**ver2.0の規定（第6条第2項）**:
```
「ガバナンストークン」とは、...国内法で定義される暗号資産に該当しないトークンであり...
```

**削除理由**: 暗号資産であること自体は問題ではなく、販売が問題となるため。

### 2.6 附則の一部

**削除された規定**:
- 設立時の資本金の額（旧第27条）
- 本店の所在場所（旧第28条）

**削除理由**: 設立時固有の事項であり、継続的な定款としては不要。

## 3. 追加された機能

### 3.1 業務執行社員会（第5章全体）

**新設内容**:
- 第20条：業務執行社員会の設置（＋規程存在時の失効条項）
- 第21条：業務執行社員会の権限
- 第22条：業務執行社員会の決議事項
- 第23条：業務執行社員会の招集及び決議方法

**追加理由**: 
- 規程がない場合のブレーキ役として機能
- DAO総会との両機関決議による意思決定の安定化
- **規程が存在する場合は失効**し、必要であれば規程で再編可能

**重要な但書き（第21条1号）**:
```
当会社のガバナンス、ファイナンス及び業務執行に関する重要な事項の決定
（ただし、DAO総会規程、トークン規程及び運営規程の制定又は変更を除く。）
```
- **規程の制定・変更は業務執行社員会の「重要な事項」に含まれない**
- これにより、DAOが規程を制定して主権を確立する道が保障される

### 3.2 業務執行社員の拒否権（第18条）

**新設内容**:
```
DAO総会規程が定められている場合、DAO総会の決議が次に定める事項に該当するときは、
業務執行社員の過半数による決議をもって、当該DAO総会の決議を差し止めることができる。
(1) 法令に違反する場合
(2) 人の生命又は身体の保護のために必要がある場合
(3) 国の機関若しくは地方公共団体又はその委託を受けた者が法令の定める事務を遂行する
    ことに対して協力する必要がある場合等のやむにやまれない必要性がある場合
(4) 当会社の定める定款又は規則に違反する場合
```

**追加理由**: 
- **規程存在時の最低限のブレーキ**として機能
- ver2.0の第19条と同様の免責事項に基づく
- 規程なし時は適用されない（業務執行社員会との両機関決議がブレーキ役）

### 3.3 DAO総会の決議事項（第16条）

**新設内容**:
```
前条第1号によりDAO総会の決定を要する重要な事項は、DAO総会規程の定めによるものとする。
DAO総会規程に定めがない又はDAO総会規程が定められていない場合は、業務執行社員会の
過半数の同意で決定するものとする。
```

**追加理由**: 
- ver2.0の第16条と同様の機能を維持
- 規程なし時のフォールバック（業務執行社員会が決定）を追加

### 3.4 規程不存在時のフォールバック規定

**追加箇所**:
| 条文 | フォールバック内容 |
|------|------------------|
| 第6条第1項 | 業務執行社員が定める方法でトークン発行 |
| 第7条第1項 | 合理的な方法で社員名簿に記入 |
| 第14条第2項 | トークンホルダーの過半数の同意で決定 |
| 第16条 | 業務執行社員会の過半数の同意で決定 |
| 第17条第1項 | 各業務執行社員が招集 |
| 第17条第2項 | トークンホルダーは一人一票、過半数同意 |
| 第25条第5項 | 出資の価額に応じて按分 |
| 第26条 | 出資の価額に応じて按分 |

**追加理由**: 規程がない状態でも定款のみで運用可能にするため。

### 3.5 代替トークン発行但書（第6条第1項）

**新設内容**:
```
但し、合理的必要性があって、業務執行社員の過半数が同意した場合は、業務執行社員が
定める方法で発行されるものを、「社員権トークン」と扱って良いものとする。
```

**追加理由**: 緊急時やトークン規程がない場合の対応。

### 3.6 持分払戻し規定（第26条第3項・第4項）

**新設内容**:
```
当会社は、DAO総会の決議によって、次に掲げる事項を定めることによって、社員権トークン
ホルダーの持分の全部又は一部を払戻すことができる。
```

**追加理由**: 社員の出口確保。金融庁確認対象。

### 3.7 議事録作成義務（第17条第3項）

**新設内容**:
```
DAO総会については、代表社員が、議事録を作成し、議事録が書面をもって作成されている
ときは、これに署名（電子署名を含む。）し、又は記名押印する。
```

**追加理由**: 運用の確実性向上。

### 3.8 規程存在時の業務執行社員会失効条項（第20条第2項）

**新設内容**:
```
ただし、DAO総会規程が定められている場合は、本章（第20条から第23条）の規定は
適用しないものとする。この場合において、DAO総会規程で業務執行社員会に関する
定めがあるときは、当該定めに従うものとする。
```

**追加理由**: 規程存在時に業務執行社員会を失効させ、必要であれば規程で再編可能にするため。

## 4. 変更された規定

### 4.1 DAO総会権限列挙

**ver2.0**: 11項目（簡潔な列挙）
**ver3.2**: 20項目（詳細な列挙）

**主な追加項目**:
- 第1号：当会社のガバナンス、ファイナンス及び業務執行に関する重要な事項の決定（包括条項）
- 第2号：金銭等の使途変更に関する信任
- 第3号：トレジャリーの使途変更に関する信任
- 第14号：運営規程の変更（トレジャリー管理規程に限定しない）

### 4.2 業務執行社員の解任方法

| ver2.0 | ver3.2 |
|--------|--------|
| DAO総会の決議**もしくは総社員の過半数の同意** | DAO総会の決議**のみ** |

**変更理由**: シンプル化。DAO総会による統一的な決定。

### 4.3 代表社員の選定方法

| ver2.0 | ver3.2 |
|--------|--------|
| DAO総会の決議によって推薦された業務執行社員を、業務執行社員の互選により選定（2段階） | 別紙社員名簿記載のとおりとする。当該規定は、DAO総会の決議により変更 |

**変更理由**: シンプル化。

### 4.4 過失要件

| ver2.0（第12条第6項） | ver3.2（第12条第5項） |
|----------------------|----------------------|
| 当該業務執行社員の故意又は**過失** | 当該業務執行社員の故意又は**重過失** |

**変更理由**: 業務執行社員のなり手確保。軽過失での責任免除を明確化。

### 4.5 出資超過分配制限の対象者

| ver2.0 | ver3.2 |
|--------|--------|
| **業務執行社員以外のトークンホルダー** | **業務執行社員（但し、当会社の事業に係る業務執行の決定について同意をするか否かの意思を表示し、かつ、当該事業の全部又は一部に従事する者に限る。)以外のトークンホルダー** |

**変更理由**: 金商法対応の明確化。実質要件を追記。

### 4.6 配当財産の種類明記

| ver2.0 | ver3.2 |
|--------|--------|
| 配当財産の種類及び帳簿価額の総額 | 配当財産の種類**（日本円、暗号資産又はトークンその他）**及び帳簿価額の総額 |

**変更理由**: DAO特性への対応。暗号資産での配当を明示。金融庁確認対象。

### 4.7 倒産等手続定義

| ver2.0 | ver3.2 |
|--------|--------|
| 破産手続**その他の法的倒産手続**開始の決定 | 破産手続開始、**民事再生手続開始、会社更生手続開始、特別清算開始若しくはこれらに類する手続**（以下「倒産等手続」という。）開始の決定 |

**変更理由**: 明確性向上。詳細列挙。

### 4.8 運営規程の制定・変更権限

| ver2.0 | ver3.2 |
|--------|--------|
| （明示的な規定なし） | DAO総会が運営規程を制定・変更可能（第12条第1項、第15条第14号） |

**変更理由**: DAO総会規程が存在する場合、すべての規程をDAO総会がデザインできるようにするため。

## 6. 金融庁確認対象事項

以下の事項については、金融庁への確認が予定されています：

1. **持分払戻し**
   - 現金等による出資に対して、暗号資産等で配当／払戻しがOKなのか
   - 特にステーブルコイン（JPYCなど電子決済手段）での払戻し可否

2. **配当財産の種類**
   - 暗号資産やトークンでの配当の法的位置づけ

## 7. 設計思想の比較

### ver2.0の設計

```
【前提】規程類が存在する

DAO総会決議 → 社員同意プロセス → （否認決議・拒否権）
```

### ver3.2の設計

```
【モード1：規程なし時】
DAO総会 ←──両機関決議──→ 業務執行社員会（第20-23条）
         （ブレーキ）

・定款のミニマムルールが適用
・業務執行社員会がセーフガードとして機能
・フォールバック規定により運用可能

【モード2：規程あり時】
DAO総会 ──→ 業務執行社員の拒否権（第18条）──→ 執行
              （最低限のブレーキ）

・業務執行社員会（第20-23条）は【失効】
・必要であれば規程で【再編】可能
・最低限のブレーキとして拒否権が有効化
・DAO総会規程がすべてをデザイン
```
