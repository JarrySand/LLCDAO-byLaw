# 定款設計分析：主権構造の観点からの過不足整理

**作成日**: 2025/12/08  
**目的**: ver3.1の設計思想との整合性確認、修正箇所の特定

---

## 1. 設計思想（あるべき姿）

### 1.1 二つのモード

```
┌─────────────────────────────────────────────────────────────────┐
│  モード1：規程が存在しない場合（不安定な状態）                    │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                    │
│  ・DAOがまだ自身をデザインできていない                            │
│  ・定款のミニマムルールが適用                                     │
│  ・業務執行社員会が「ブレーキ」として機能                         │
│  ・重要事項は両機関の決議が必要（セーフガード）                   │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│  モード2：規程が存在する場合（安定した状態）                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                    │
│  ・DAO自身がDAOをデザインできている                               │
│  ・定款で規定される業務執行社員会は【失効】                       │
│  ・必要であれば規程で業務執行社員会を【再編】可能                 │
│  ・最低限のブレーキとして、業務執行社員による【拒否権】が有効化   │
│  ・ver2.0と同様の機能を実現可能                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 核心的なポイント

| 状況 | 意思決定の構造 | ブレーキ機構 |
|------|---------------|-------------|
| **規程なし** | 定款のミニマムルール（両機関決議） | **業務執行社員会**（セーフガード） |
| **規程あり** | DAO総会規程がすべてをデザイン | **業務執行社員の拒否権**（最低限のブレーキ） |

### 1.3 設計の意図

```
【設立初期】規程なし                     【成熟期】規程あり
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                                   
  DAO総会 ←──両機関決議──→ 業務執行社員会      DAO総会規程がすべてをデザイン
           （ブレーキ）                         ├─ DAO総会の権限
                                               ├─ 業務執行社員会は【失効】
                                               │   （規程で再編可能）
                                               └─ 業務執行社員の【拒否権】が有効
                                                   （最低限のブレーキ）
```

### 1.4 拒否権の位置づけ

- **規程なし時**：業務執行社員会との両機関決議がブレーキ役
- **規程あり時**：業務執行社員会は失効するが、最低限のセーフガードとして業務執行社員の拒否権が有効化

```
【規程あり時の拒否権】
DAO総会決議 ──→ 業務執行社員による拒否権チェック ──→ 執行

※ver2.0の「社員同意・否認決議・拒否権」に代わる最低限のブレーキ
```

## 2. 規程の体系と制定権限

### 2.1 規程類の一覧

| 規程 | 役割 | 制定・変更権限 |
|------|------|---------------|
| **DAO総会規程** | DAOのガバナンス全体をデザイン | DAO総会 |
| **トークン規程** | トークンの発行・移転等を規定 | DAO総会 |
| **運営規程** | 業務執行の詳細を規定 | ？ |
| **トレジャリー管理規程** | 資金管理を規定 | DAO総会 |

### 2.2 問題点：運営規程の制定権限

**ver3.1の現状：**
- 第12条第1項：「**業務執行社員会において**別途定める運営規程」
- 第15条第12号：「運営規程（**トレジャリー管理規程に限る**。）の変更」がDAO総会権限
- 第19条第9号：「運営規程の変更（トレジャリー管理規程は除く。）」が業務執行社員会権限

**問題：**
- 運営規程の変更権限が業務執行社員会にある
- DAO総会が「業務執行社員会の権限をデザインする」ためには、運営規程もDAO総会が制定・変更できる必要がある

**あるべき姿：**
- 規程がある場合は、DAO総会規程が運営規程も含めてすべてをデザイン
- 規程がない場合は、業務執行社員会が暫定的に運営を決定

## 4. 修正箇所サマリー

### 4.1 最高優先度（設計の根幹）

| # | 条文 | 修正内容 | 理由 |
|---|------|----------|------|
| 1 | **第15条** | 権限列挙に「**当会社のガバナンス、ファイナンス及び業務執行に関する重要な事項の決定**」を**追加** | DAOが自己規定的に権限を解釈できるようにする |
| 2 | **第16条（新設）** | 「**DAO総会の決議事項**」を追加：「前条第●号によりDAO総会の決定を要する重要な事項は、DAO総会規程の定めによるものとする」 | DAO総会規程で「重要な事項」を自己定義可能にする |

**これが欠けている問題点：**
- 現状ver3.1の第15条は具体的な18項目のみを列挙
- 「ガバナンス、ファイナンス及び業務執行に関する重要な事項」という包括条項がない
- ver2.0の第16条に相当する「決議事項をDAO総会規程で定める」規定がない
- **結果：DAO総会規程でDAOが自己規定的に権限をデザインできない**

### 4.2 高優先度（運営規程の権限）

| # | 条文 | 修正内容 | 理由 |
|---|------|----------|------|
| 3 | **第15条** | 「運営規程（トレジャリー管理規程に限る。）」→「**運営規程**」 | DAO総会が運営規程を制定・変更できるようにする |
| 4 | **第19条** | 「運営規程の変更（トレジャリー管理規程は除く。）」を**削除** | 運営規程の変更権限をDAO総会に一本化 |
| 5 | **第12条第1項** | 「業務執行社員会において」→「**DAO総会又は業務執行社員会において**」 | 運営規程の制定主体を柔軟に |

### 4.3 中優先度（フォールバック・柔軟性）

| # | 条文 | 修正内容 | 理由 |
|---|------|----------|------|
| 6 | **第14条第2項** | 追加：「DAO総会規程で規定されていない事項については、トークンホルダーの過半数の同意で決定」 | 規程なし時のデフォルト決議方法を明確化 |
| 7 | **第15条柱書** | 追加：「ただし、DAO総会規程に別段の定めがある場合はこの限りでない」 | 規程あり時に両機関決議を変更可能にする |

## 5. 自己規定的権限解釈の仕組み

### 5.1 ver2.0の構造（参考）

```
第15条（DAO総会の権限）
├─ 第1号：「当会社のガバナンス、ファイナンス及び業務執行に関する重要な事項の決定」
│        ↑ 包括条項（DAOの権限範囲を広く宣言）
├─ 第2号〜第11号：具体的な権限列挙
└─ ...

第16条（DAO総会の決議事項）
「前条第1号によりDAO総会の決定を要する重要な事項は、DAO総会規程の定めによる」
        ↑ 「重要な事項」の具体的内容をDAO総会規程で自己定義可能にする
```

**効果：**
- 第15条第1号で広範な権限を宣言
- 第16条で「重要な事項」の内容をDAO総会規程に委任
- DAO総会規程で「何が重要な事項か」を自己規定的に定義できる
- **DAOが自らのガバナンスをデザインする自己決定権を確保**

### 5.2 ver3.1の現状の問題

```
第15条（DAO総会の権限）
├─ 第1号：社員が払い込んだ金銭等の使途変更
├─ 第2号：トレジャリーの使途変更
├─ ...
└─ 第18号：前各号の検討契約締結

【問題】包括条項がない → 列挙された18項目に限定される
【問題】第16条がない → DAO総会規程で権限を拡張できない
```

### 5.3 修正後の構造

```
第15条（DAO総会の権限）
├─ 【追加】第1号：「当会社のガバナンス、ファイナンス及び業務執行に関する重要な事項の決定」
├─ 第2号〜：具体的な権限列挙（既存の18項目）
└─ ...

【追加】第16条（DAO総会の決議事項）
「前条第1号によりDAO総会の決定を要する重要な事項は、DAO総会規程の定めによる。
 DAO総会規程に定めがない又はDAO総会規程が定められていない場合は、
 業務執行社員会の過半数の同意で決定する。」
```

**効果：**
- 規程あり時：DAO総会規程で「重要な事項」を自己定義、すべてをデザイン
- 規程なし時：業務執行社員会が暫定的に決定（ブレーキ機能）

## 6. 修正による効果

### 6.1 規程がない場合（修正後）

```
【定款のミニマムルール】

DAO総会 ←──両機関決議──→ 業務執行社員会
         （ブレーキ）

・重要事項は両機関の決議が必要
・業務執行社員会がセーフガードとして機能
・不安定な状態でも意思決定が可能
```

### 6.2 規程がある場合（修正後）

```
【DAO総会規程がすべてをデザイン】

DAO総会規程で定める構造
├─ DAO総会の権限範囲
├─ 業務執行社員会の権限範囲（対等も可、限定も可）
├─ 両者の関係（両機関決議の要否）
├─ 運営規程の内容
└─ その他すべて

※「DAO総会規程に別段の定めがある場合はこの限りでない」により
  定款のミニマムルール（両機関決議）を上書き可能
```

### 6.3 ver2.0との機能等価性

| 機能 | ver2.0 | ver3.1（修正後・規程あり） | 実現方法 |
|------|--------|--------------------------|----------|
| DAO総会主権 | ✅ | ✅ | DAO総会規程で業務執行社員会権限を限定 |
| 対等な構造 | - | ✅ | DAO総会規程で両機関決議を維持 |
| 業務執行社員会優位 | - | ✅ | DAO総会規程で業務執行社員会に広い権限委任 |
| 社員同意プロセス | ✅ | 不要 | - |
| 否認決議・拒否権 | ✅ | 不要 | 両機関決議がブレーキ役を代替 |
