# 用語集 (Glossary)

本用語集は、合同会社型DAOの定款、各規程、および運用ガイドで使用される主要な用語を定義・解説するものです。

## 1. 組織・役割

| 用語 | 定義・解説 |
| --- | --- |
| **本DAO** | 定款に基づき設立された「●●DAO合同会社」のこと。 |
| **社員** | 会社法上の社員のこと。出資を行い、定款の定めに従い権利義務を有する。本DAOでは「業務執行社員」と「非業務執行社員」に大別される。 |
| **業務執行社員** | 本DAOの業務を執行する権限と責任を持つ社員。実名で登記され、法令上の法的責任（無限責任等ではないが、任務懈怠責任等）を負う可能性がある。DAO総会の決議に基づき業務を行う。 |
| **非業務執行社員** | 業務執行権を持たない社員。主に「社員権トークン」を保有するDAOメンバーがこれに該当する。原則として有限責任であり、匿名性が確保される（社員名簿は業務執行社員のみが管理）。 |
| **DAOメンバー** | 一般的に、社員権トークンを保有し、本DAOに参加している者（非業務執行社員）を指す。 |
| **代表社員** | 業務執行社員の中から選定され、会社を代表する権限を持つ者。 |
| **署名担当代表社員** | 代表社員のうち、特にDAOの代表署名や代表印の管理・使用を担当する者。 |
| **財務担当業務執行社員** | トレジャリー管理規程に基づき、資金管理（送金承認等）を担当する業務執行社員。 |
| **経理担当業務執行社員** | トレジャリー管理規程に基づき、記帳や会計処理を担当する業務執行社員。 |
| **運営責任者** | ガイド上の役割の一つ。オペレーションリードとして、年間計画の策定やプロセスの遵守管理を行う。 |
| **議長** | DAO総会の進行や議事管理を行う役割。 |

## 2. トークン・権利

| 用語 | 定義・解説 |
| --- | --- |
| **社員権トークン** | 本DAOの社員（非業務執行社員）たる地位を表章する**非代替性トークン (NFT)**。これを保有することで、社員としての権利（DAO総会での議決権等）を得る。譲渡には制限がある場合がある。 |
| **ガバナンストークン** | 本DAOが発行する**非代替性トークン (NFT)**のうち、社員としての地位は伴わないが、DAO総会における議決権が付与されたもの。 |
| **リワードトークン** | 本DAOへの貢献に対する報酬として発行される**代替性トークン (FT)**。議決権は持たないが、特定の便益との交換や、条件付きでガバナンストークンへの交換（Burn & Mint等）が可能な場合がある。 |
| **SBT (Soulbound Token)** | 譲渡不可能なトークン。本DAOでは、特定の資格や貢献証明として利用される場合がある。 |
| **トークンホルダー** | 広義には本DAOが発行するいずれかのトークンを保有する者。狭義（定款・総会規程上の「議決権保有者」）には、社員権トークンまたはガバナンストークンの保有者を指す。 |
| **議決権保有者** | DAO総会において議決権を行使できる者。具体的には、社員権トークンホルダーおよびガバナンストークンホルダーの総称。 |
| **ベスティング (Vesting)** | トークンを一定期間ロックし、時間の経過や条件達成に応じて徐々に権利を確定・付与する仕組み。 |
| **クリフ (Cliff)** | ベスティングにおいて、権利確定が開始されるまでの待機期間。この期間中はトークンは一切付与されない。 |
| **ミント (Mint)** | スマートコントラクトを通じて、新たにトークンを発行すること。 |
| **バーン (Burn)** | トークンを特定の操作により永久に削除（消却）すること。総発行量を減らすために行われる。 |
| **メタデータ (Metadata)** | トークンに付随するデータ。NFTの場合、画像データへのリンクや属性情報などが含まれる。 |

## 3. 会議体・意思決定

| 用語 | 定義・解説 |
| --- | --- |
| **DAO総会** | トークンホルダー（議決権保有者）全員で構成される、本DAOの最高意思決定機関。ブロックチェーン上の投票システムやDiscord等のツールを用いて開催・決議される。 |
| **重要決議** | DAOの存続、分散性の維持、定款変更、多額の資金移動など、DAOにとって特に重要な事項に関する決議。高い可決要件（定足数・賛成率）が設定される。 |
| **通常決議** | DAOの一般的な運営事項、予算承認、細則の制定などに関する決議。 |
| **その他の決議** | 委任範囲内の細目や、重要度・緊急度が比較的低い事項に関する決議。 |
| **定足数 (Quorum)** | 決議が成立するために最低限必要な投票数（または投票権の割合）。 |
| **拒否権** | 業務執行社員が、DAO総会の決議が法令違反やDAOの存続を脅かす場合等の「免責事項」に該当すると判断した際に、その決議の執行を差し止める権利（定款第19条）。 |
| **否認決議** | 法的要請等に基づき、社員（業務執行社員含む）がDAO総会の議決を否認する決議（定款第18条）。 |
| **社員同意** | 会社法上、社員の同意が必要な事項（定款変更等）について、DAO総会での議決内容に基づき、形式的に社員が行う同意手続き。 |
| **提案 (Proposal)** | 意思決定のために提出される議案。アイデア出し、ドラフト作成、賛同集めを経て正式な投票にかけられる。 |
| **ドラフト (Draft)** | 正式な投票にかける前の議案の草案。フォーラム等で議論・修正される。 |
| **Snapshot** | ガス代不要で投票ができるオフチェーン投票プラットフォーム。本DAOでは主な投票ツールとして利用される。 |
| **オフチェーン投票** | ブロックチェーン上に記録しない投票方式。Snapshot等がこれにあたり、ガス代がかからないメリットがある。 |
| **オンチェーン投票** | ブロックチェーン上に直接記録する投票方式。透明性と改ざん耐性が高いが、ガス代（手数料）が発生する。 |

## 4. 業務執行・運営・技術

| 用語 | 定義・解説 |
| --- | --- |
| **トレジャリー** | 本DAOが保有・管理する資金（暗号資産、法定通貨等）およびそれを保管するウォレットの総称。 |
| **ウォレット** | 暗号資産やトークンを保管・管理し、署名を行うためのソフトウェア（MetaMask等）。DAOにおいては、個人の認証手段（ID）としての機能も果たす。 |
| **アドレス** | ブロックチェーン上の口座番号のような文字列。ユーザーの「ウォレットアドレス」や、プログラムの場所を示す「コントラクトアドレス」がある。 |
| **スマートコントラクト** | ブロックチェーン上で自動的に契約や処理を実行するプログラム。トークンの発行、投票、資金管理などに使用される。 |
| **マルチシグ (Multi-sig)** | 資金移動などの際に、複数の署名（承認）を必要とする仕組み。トレジャリーのセキュリティを高めるために用いられる。 |
| **署名権者 (Signer)** | マルチシグウォレットにおいて、取引の承認（署名）を行う権限を持つ者。通常は業務執行社員が務める。 |
| **閾値 (Threshold)** | 何かを決定・実行するために必要な最低数。マルチシグの署名数や、投票の可決ラインなどを指す。 |
| **トランザクション (Transaction)** | ブロックチェーン上での取引データ。送金やスマートコントラクトの実行などがこれにあたる。 |
| **ガス代 (Gas Fee)** | ブロックチェーン（Ethereum等）を利用する際に発生する手数料。ネットワークの混雑状況により変動する。 |
| **KYC (Know Your Customer)** | 本人確認手続き。トークンの取得や特定の権利行使（リワードの交換等）の際に求められる場合がある。 |
| **Sybil攻撃** | 一人の人間が多数のアカウント（ウォレット）を作成し、不当に多数の票を投じる等の攻撃行為。 |
| **GOX (ゴックス)** | 誤送金やハッキング、秘密鍵の紛失等により、暗号資産を永久に失うこと。 |
| **インシデント (Incident)** | 運用上のトラブル、事故、不正行為などの予期せぬ事象。重要度に応じてレベル分けされ対応される。 |
| **免責** | DAO総会の決議に基づき業務を執行した結果、DAOに損害が生じた場合でも、業務執行社員の故意・重過失がない限り、その責任を問わないこと。 |
| **別紙** | 各規程に付随し、頻繁に変更される可能性のある詳細事項（ツール名、パラメータ、具体的な数値等）を記載した文書。「共通運用.md」「DAO総会別紙.md」「トークン別紙.md」などがある。 |

## 5. 文書体系

| 用語 | 定義・解説 |
| --- | --- |
| **定款** | 本DAOの根本規則。会社法に基づき作成され、公証人の認証および登記が必要。最も強い効力を持つ。 |
| **DAO総会規程** | DAO総会の運営、決議方法、議案の提案プロセス等を定めた規程。 |
| **運営規程** | 業務執行社員による業務執行の体制、プロセス、情報開示等を定めた規程。 |
| **トークン規程** | 各種トークンの発行条件、権利内容、禁止事項等を定めた規程。 |
| **ガイド (Guide)** | 定款や規程の解釈、運用手順、ベストプラクティスをまとめた解説書。法的拘束力はないが、実務上の指針となる。 |
